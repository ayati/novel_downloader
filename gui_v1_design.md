# novel_downloader GUI v1 設計書

> Windows 向け GUI 皮（フロントエンド）の v1 設計。
> 本書は「ここまでに決まったこと」を記録する基盤ドキュメント。詳細仕様は順次追記する。

最終更新: 2026-06-23

---

## 1. 目的とゴール

CLI ツール `novel_downloader.py`（青空文庫テキスト／縦書き ePub3 を生成）を、
**コマンドラインを使わない一般ユーザー**がそのまま使える Windows GUI で包む。

v1 の達成条件（＝ユーザーがゴールを感じる瞬間）:

> **お目当ての小説の ePub をダウンロードし、エクスプローラでそのファイルが見られる。**

---

## 2. 想定ユーザー像

- PC の基本操作はできるが、**コマンドラインは使わない**。
- ブラウザで読みたい小説のページを開くことはできる。
- **マニュアルを読まない。** GUI を見て、その場で操作を理解したい。
- 「ePub がダウンロードできてエクスプローラで見られれば、最初の目的は達成」と感じる。

設計上の最優先指針: **ブラウザ↔フォルダ間の摩擦をゼロに近づける。**

---

## 3. 確定した基盤方針

| 項目 | 採用 | 理由（要約） |
|---|---|---|
| ① エンジン統合方式 | **A: サブプロセス方式** | 本体コードを無改修で再利用。`.exe` 資産をそのまま活用。クラッシュがGUIに波及しない。中断が容易 |
| ② GUI フレームワーク | **CustomTkinter** | 1画面の最小構成に最適。軽量・部品が少なく exe が肥大しない。見た目も今風 |
| ③ UX 形態 | **単発ダウンロードフォーム（徹底ミニマル）** | 「URL欄＋大ボタン」だけを表に出し、全オプションは詳細設定の奥へ |

### 3.1 ユーザー像に直結する 4 つの追加要素（v1 必須）

1. **完了時にエクスプローラを自動で開く（＋その .epub を選択状態に）**
   `explorer /select,"...\xxx.epub"` 相当。ゴールそのものを保証する最重要機能。
2. **起動時にクリップボードの URL を自動入力**
   対応サイトの URL ならURL欄を自動で埋め、大ボタンを強調。実質ワンクリック。
   （自動実行はせず、確定はユーザーのクリックに委ねる。）
3. **出力先を聞かない（固定の分かりやすい既定フォルダ）**
   既定は **`ダウンロード\小説`**（`%USERPROFILE%\Downloads\小説`）。初回に保存先選択を求めない。変えたい人だけ詳細設定で。
   フォルダ名は日本語「小説」で確定（非技術者には母語名が最も認識しやすい）。
4. **生ログを見せない／やさしい日本語のエラー**
   表は「進捗バー＋"第N話/全M話を取得中…"＋完了！」のみ。失敗時もトレースではなく平易な文言。生ログは「詳細」の奥に。

## 4. 画面イメージ（状態遷移）

最小構成: **1画面・主要4状態**。

### 状態1: 起動直後（クリップボードに対応URLあり＝最頻ケース）
```
📖 小説ePubダウンローダー
  小説のURLを貼り付け
  [ https://ncode.syosetu.com/n0022gd/ ]   ← 自動入力済み
        【 ⬇ ダウンロード 】               ← 大ボタン（強調）
  保存先： ダウンロード\小説                   ← 表示のみ・小さく
  ▸ 詳細設定                                ← 折りたたみ（閉）
```

### 状態2: 起動直後（URLなし）
URL欄はプレースホルダ表示、大ボタンはグレーアウト（URL入力で活性化）。

### 状態3: ダウンロード中
```
  [ https://... ]  ← 編集ロック
        【 ⏸ 中止 】                       ← ボタンが「中止」に変化
  取得中…  第 12 話 / 全 30 話
  ████████████░░░░░░░░  40%
```
進捗は subprocess の stdout 行から抽出（①A方式）。生ログは出さない。

### 状態4: 完了（達成の瞬間）
```
  ✅ 完了しました！
  「無職転生」.epub を保存しました
        【 📂 フォルダを開く 】              ← 保険。完了と同時に自動で開く
```
完了と同時にエクスプローラが開き、.epub が選択状態になっている。

### 状態5: エラー（未対応サイト等）
```
  ⚠ うまくいきませんでした
  このURLは対応していないサイトかもしれません。
  URLを確認してもう一度お試しください。
        【 ⬇ もう一度 】
  ▸ 詳細を表示                              ← 困った時だけ生ログ
```
※ 文言・ボタンは §8 を正とする（①未対応サイト〔＋［対応サイトを見る］〕／②ハーメルン／③DL中失敗で出し分け）。上図は概念図。

## 5. 詳細設定パネル（開いたときのデザイン）

普段は閉じている。開いても**非破壊**（既定値が選択済みで、見て閉じても何も壊れない）。

並び順は CLI フラグ順ではなく**ユーザーにとっての意味・印象の強さ順**:
**保存先 → 表紙 → （区切り線）→ 普段は変更不要なもの**。

```
▾ 詳細設定

  保存先
  [ ダウンロード\小説 ] [変更]

  表紙（ePubの“顔”）
  ◉ おまかせ（自動で作る）          ← 既定
  ○ サイトの公式表紙を使う          (--use-site-cover)
  ○ 自分の画像を選ぶ…             (--cover-image)
       [プレビュー] [ 画像が未選択です ] [画像を選ぶ]

  ───────  ここから下は普段は変更不要  ───────

  ☐ 横書きにする (--horizontal)   ☐ Kobo端末向け .kepub.epub (--kobo)
  ☐ 目次を本の最後に置く (--toc-at-end)
  本文のフォント： 標準（埋め込みなし） [選ぶ][標準に戻す] (--font)
  取得間隔 [1.5]秒 (--delay)       文字コード [UTF-8 ▾] (--encoding)
```

設計判断:
- **表紙は排他ラジオ**（チェックボックス3つにしない）。`--cover-image` が `--use-site-cover` より優先という隠れ仕様を、UIの形で**矛盾を作れなく**する。
- CLI フラグ名は主役にしない（淡い補足／ツールチップ程度）。
- 「自分の画像」選択時のみファイル選択を活性化。
- 非対応画像は本体仕様どおり自動生成へフォールバック。UI はやさしく通知し、エラーで止めない。
- **詳細設定の内部も2層化**: 上段（保存先・表紙＝意味が分かる）／区切り線／下段（技術的・淡め）。
- **前回の選択を記憶**（保存先・表紙）。次回起動時に復元。

## 6. データフロー（裏側）

```
起動
 └▶ クリップボードを読む
      ├ 対応サイトURL? ─Yes▶ URL欄へ自動入力＋ボタン強調
      └           ─No ▶ 空のまま待機
（ユーザーが「ダウンロード」を押す）
 └▶ URLを簡易検証（detect_site相当）
      └ 未対応なら即やさしいエラー
 └▶ subprocess で本体起動（①A方式）
      novel_downloader.exe <URL> --output-dir "%USERPROFILE%\Downloads\小説"
      ＋ Windows: CREATE_NO_WINDOW（黒窓を出さない）
      ＋ PYTHONUNBUFFERED=1（進捗をリアルタイムに受け取る／§9.3）
      ＋ 別スレッド実行（GUIを固めない）
 └▶ stdout を1行ずつ読む
      ├ 進捗行 `[N/M]` → "第N話/全M話" + バー（§9）
      └ `✅ ePub出力完了: <path>` 行を捕捉して epub パスを保持
        UI更新は queue 経由でGUIスレッドへ安全に渡す
 └▶ プロセス終了
      ├ returncode==0 ▶ 捕捉した epub パスで explorer /select ▶ 「✅完了」
      │                （未捕捉なら出力先の最新 .epub にフォールバック）
      └ returncode!=0 ▶ 「⚠ うまくいきませんでした」＋生ログは“詳細”の奥
```

肝:
- **GUI ↔ エンジンは完全分離**。本体コードは無改修。
- subprocess は別スレッド、UI 更新は `queue` 経由（Tkinter 定番）。
- 完了イベントは **returncode==0**、開く対象の **epub パスは `✅ ePub出力完了:` 行**から取得（役割分担／§9.2）。これが「フォルダを開いて当該 epub を選択」の信頼性を担保する。
- 「中止」は実行中プロセスを `terminate()`。中断時の中間 `.txt` は **残す**（再開に使える・v1 既定）。

## 7. 未対応サイトの扱い（案Y: 本体に判定モードを追加）

方針: **サイト判定の正は本体ひとつに集約する。** GUI に対応サイトのフルリストを複製しない
（このプロジェクトはスクレイパーが増え続けるため、GUI 側に判定を置くと確実にドリフトする）。

### 7.1 本体に読み取り専用モード `--detect-site URL` を追加
①A「本体無改修」への**唯一の小さな例外**。読み取り専用・ネットワーク不要・他用途でも再利用可。
- 既存 `detect_site()` / `normalize_url()` を流用し、結果を1行で出力して即終了。**ダウンロードはしない。**
- **短縮URLは展開しない**（オフライン・即時の約束を守るため）。短縮URLは `site:null` 扱いになり自動入力は効かないが、実ダウンロード時に本体の `expand_short_url` が展開するので実害は「自動入力の取りこぼし」のみ。

確定スキーマ（JSON 1行）:
```json
{ "schema": 1,
  "site": "narou",            // 未対応は null
  "display_name": "小説家になろう",
  "needs_playwright": false,  // ハーメルンは true
  "normalized_url": "https://ncode.syosetu.com/n0022gd/" }
```

| キー | 内容 |
|---|---|
| `schema` | 出力フォーマットの版番号（CLI↔GUI を別ビルド配布するための互換判定）。v1 は `1` |
| `site` | サイトID（`_SITE_DISPATCH` のキー）。未対応は `null` |
| `display_name` | 表示名。確認表示・表紙の底本リンクラベル（「カクヨムで読む」等）に使う |
| `needs_playwright` | `true`（ハーメルン）なら GUI は DL 前に早期ブロック。将来 playwright 導入時は解禁可 |
| `normalized_url` | 話数URL→作品トップに正規化済みのURL（未対応時は省略 or `null`） |

### 7.2 GUI の使いどころ（一般未対応もハーメルンも同じ1コマンドで分岐）
1. **クリップボード自動入力**: 起動時、クリップボードURLを `--detect-site` で判定。
   `site` が非 null **かつ** `needs_playwright==false` のときだけ自動入力＋ボタン強調。
2. **ダウンロード前チェック**:
   - `site==null` → 「対応していないサイトかもしれません」（一般の未対応サイト）
   - `needs_playwright==true`（ハーメルン）→ **ダウンロード開始前に早期に弾く**。
     「このアプリではハーメルンは未対応です」。深い失敗（DL途中で Chromium 不在エラー）を防ぐ。
     ※ 将来 playwright を同梱／導入したら、同じ `needs_playwright` フラグを見て解禁できる。

→ GUI 側に判定ロジック・サイトリストを一切持たず、未対応サイトもハーメルンも本体の判定結果だけで分岐する。

※ クリップボード判定（`--detect-site`）は**別スレッドで実行**し、ウィンドウは即時表示する（判定が返り次第あとから自動入力）。CLI exe が onefile の場合の spawn 起動コスト（§12.5）で UI を固めないため。

### 7.3 本体に読み取り専用モード `--list-sites` を追加
対応サイト一覧を出すための、もう一つの読み取り専用モード（`--detect-site` と同じ「本体が正」の思想）。
GUI／README に対応サイトリストを二重持ちさせない＝スクレイパー追加でドリフトさせないため、**本体に一覧を出させる**。
- `_SITE_DISPATCH` を列挙して JSON で出力し即終了。**オフライン・即時。**
- 出力（案）:
```json
[ {"site":"narou","display_name":"小説家になろう"},
  {"site":"kakuyomu","display_name":"カクヨム"}, ... ]
```
- GUI の用途: エラー①（未対応サイト）の **［対応サイトを見る］** が押されたとき一度だけ呼び、
  **日本語名の一覧をアプリ内に表示**。本体にスクレイパーが増えれば一覧も自動追従＝GUI も README も無改修。

### 7.4 本体側に必要な追加（GUI のための読み取り専用モード）
①A「本体無改修」の例外は、いずれも**読み取り専用・ネットワーク不要・他用途でも再利用可**な小さな2モードのみ:

| モード | 用途 | 出力 |
|---|---|---|
| `--detect-site URL` | サイト判定・正規化・ハーメルン早期ブロック | JSON 1行（§7.1） |
| `--list-sites` | 対応サイト一覧（［対応サイトを見る］） | JSON 配列（§7.3） |

## 8. エラーメッセージ文言

トーン方針: 件名は短く記号は `⚠` のみ。**フラグ名・例外名・コードは一切出さない。**
原因をユーザーのミスと断定しない。**必ず次の一手**を書く。生ログは常に「詳細を表示」の奥。

### ① 未対応サイト（`site == null`）
```
⚠ このサイトには対応していません

このURLの小説サイトには、このアプリは対応していません。
URLが正しいか、対応しているサイトかをご確認ください。

[ 対応サイトを見る ]            [ ⬇ もう一度 ]
```
- ［対応サイトを見る］は §7.3 `--list-sites` の結果をアプリ内に一覧表示（GUI にサイト名を直書きしない）。

### ② ハーメルン（`needs_playwright == true`）
```
⚠ ハーメルンには対応していません

ハーメルンの小説は、このアプリではダウンロードできません。
申し訳ありませんが、別のサイトのURLでお試しください。

[ ⬇ もう一度 ]
```
- **サイト名を名指し**するのが肝（「なぜか失敗」ではなく「ハーメルンだから」と理解できる）。
- playwright 等の技術理由は書かない（このユーザーには無意味で不安を招く）。

### ③ 対応サイトなのに DL 中に失敗（ネットワーク・サイト構造変化など）
①②（DL 前の事前チェック）とは別カテゴリ。**DL 開始後の失敗**用。
```
⚠ うまくいきませんでした

ダウンロード中に問題が発生しました。
通信状態を確認して、もう一度お試しください。
それでも繰り返す場合は、しばらく時間をおいてください。

[ ⬇ もう一度 ]      ▸ 詳細を表示
```
- 原因を断定できないので中立に。［詳細を表示］で生ログ（困った人だけが開く）。

## 9. stdout → 進捗表示の対応表（§4 状態3 の根拠）

本体の stdout を解析して進捗バー／状態テキストを更新する（①A 方式）。**サイト別分岐は不要。**

### 9.1 進捗行フォーマット（全スクレイパー共通）
1話（または1章・1ページ）取得ごとに、ほぼ全サイトが同型で出力する:
```
  [   12/345] 第十二話 タイトル…      ← 大多数（話単位）
  [  3/120] page=3                    ← ハーメルン（ページ単位）
  [ 5/40] 第五章（p.10–18）           ← 野いちご / berry's（章単位）
```
**各行に「現在 N / 総数 M」が両方入る** → GUI は別途「総数」行を探す必要がない。
`M` は実際に取得する数（resume 時は差分数）＝進捗バーの分母として正確。

### 9.2 抽出（正規表現1つで足りる）
```
^\s+\[\s*(\d+)\s*/\s*(\d+)\s*\]\s*(.*)$
        └現在 N    └総数 M       └タイトル
```

| 取得 | GUI 反映 |
|---|---|
| `N` / `M` | 進捗バー = N/M、テキスト「第 N 話 / 全 M 話 を取得中…」 |
| タイトル | （任意）取得中の話タイトル |
| 最初の `[N/M]` が来る前 | 「準備中…」（作品情報・目次取得フェーズ）＝不確定アニメ |
| **完了イベント** | **returncode==0 で判定**（堅い）。完了の有無は stdout 解析に依存しない |
| **出力 epub パス** | `✅ ePub出力完了: <path>` 行を捕捉して取得（`explorer /select` 用）。未捕捉時は出力先の最新 `.epub` にフォールバック（§6） |

### 9.3 ハマりどころ（2点だけ注意）
1. **ステージ見出し `[1/3]` との衝突**: カクヨム等は `[1/3] 作品情報を取得中` も出力する。
   - 1話進捗行は必ず**先頭に空白**（`  [`）、ステージ見出しは**行頭（空白なし）**。
   - → 正規表現の `^\s+`（先頭空白必須）がそのまま安全フィルタになり `[1/3]` を弾く。
2. **出力バッファリング**: Python はパイプ出力時に stdout をまとめてバッファするため、対策しないと進捗が最後に一括で出てバーが動かない（print に `flush=True` なし）。
   - → subprocess 起動時に **`PYTHONUNBUFFERED=1`** を環境変数で渡す（`.exe` / `.py` 共通で有効。`.py` なら `-u` でも可）。**ライブ進捗の成否はこの一点。**

### 9.4 堅牢性
完了は returncode で確定するため、stdout 解析が多少こけてもバーが飛ぶ程度で致命傷にならない。
必須の前提は `PYTHONUNBUFFERED=1` のみ。

## 10. ウィジェット構成の対応表

凡例: 「表示」= §4/§5 の画面要素、「CTk部品」= CustomTkinter ウィジェット、「CLI引数/動作」= 生成する引数 or GUI動作。
**ファイル/フォルダ選択ダイアログは CustomTkinter に無いため `tkinter.filedialog` を併用する。**

### 10.1 メインウィンドウ
| 表示 | CTk部品 | CLI引数 / 動作 |
|---|---|---|
| URL入力欄 | `CTkEntry` | 位置引数 `url`（起動時クリップボードから自動入力／§7.2） |
| ダウンロード（大ボタン） | `CTkButton` | subprocess 起動。URL空なら `disabled` |
| 中止（DL中の同ボタン） | 同 `CTkButton`（文言・色を差し替え） | 実行中プロセスを `terminate()` |
| 保存先表示（小） | `CTkLabel` | `--output-dir` の現在値を表示のみ |
| 進捗バー | `CTkProgressBar` | stdout `N/M`→`set(N/M)`。準備中は `mode="indeterminate"`+`start()`（§9） |
| 進捗テキスト | `CTkLabel` | 「第 N 話 / 全 M 話 を取得中…」（§9.2） |
| 完了表示 | `CTkLabel` | returncode==0 で「✅ 完了しました！」（§9.2） |
| フォルダを開く | `CTkButton` | `explorer /select,<epub>`（完了時に自動実行＋保険ボタン） |
| エラー表示 | `CTkLabel`（必要なら枠 `CTkFrame`） | §8 ①②③ の文言 |
| 詳細を表示（生ログ） | `CTkButton`（トグル）→ `CTkTextbox` | 取得した stdout/stderr 全文。既定は隠す |
| 詳細設定（折りたたみ） | `CTkButton`（▸/▾）→ `CTkFrame` 表示切替 | §10.2 を開閉（`grid()` / `grid_remove()`） |

### 10.2 詳細設定パネル（§5）
| 表示 | CTk部品 | CLI引数 / 動作 |
|---|---|---|
| 保存先 [path] | `CTkEntry`（読み取り主体） | `--output-dir`（既定 `%USERPROFILE%\Downloads\小説`） |
| 保存先 [変更] | `CTkButton`→`filedialog.askdirectory` | 上記 Entry を更新 |
| 表紙：おまかせ | `CTkRadioButton`（共有 `StringVar`=`"auto"`） | フラグなし（自動生成） |
| 表紙：サイトの公式表紙 | `CTkRadioButton`（=`"site"`） | `--use-site-cover` |
| 表紙：自分の画像 | `CTkRadioButton`（=`"file"`） | `--cover-image <FILE>` |
| └ 画像を選ぶ | `CTkButton`→`filedialog.askopenfilename` | 選択パスを保持。`"file"` 時のみ活性 |
| └ ファイル名表示 | `CTkLabel` | 選択ファイル名 or「未選択」 |
| └ プレビュー | `CTkLabel(image=CTkImage)` | 縮小表示（**Pillow 必要**。無ければ省略） |
| 区切り線 | `CTkFrame`（高さ1px） | 視覚的な段差（§5 二層化） |
| 横書きにする | `CTkCheckBox` | on→`--horizontal` |
| Kobo端末向け | `CTkCheckBox` | on→`--kobo` |
| 目次を本の最後に置く | `CTkCheckBox` | on→`--toc-at-end` |
| 本文のフォント | `CTkLabel`＋`CTkButton`（選ぶ→`filedialog`／標準に戻す） | `--font <FILE>`（未選択は無し。ファイル欠落は埋め込みなしへフォールバック） |
| 取得間隔（秒） | `CTkEntry`（数値・既定1.5） | `--delay <SEC>` |
| 文字コード | `CTkOptionMenu` | `--encoding <ENC>`（utf-8 / utf-8-sig / shift_jis / cp932） |

排他制御: 表紙の3つは**1つの `StringVar` を共有するラジオ**＝同時選択不可。
`--cover-image` と `--use-site-cover` が両立しない本体仕様を、UI の形状で保証する（§5）。

### 10.3 ダイアログ・補助
| 表示 | CTk部品 | 動作 |
|---|---|---|
| 対応サイト一覧（[対応サイトを見る]） | `CTkToplevel` + `CTkScrollableFrame` + `CTkLabel`×N | `--list-sites`（§7.3）の `display_name` を一覧表示 |
| 生ログ表示 | `CTkTextbox`（§10.1 のトグル先） | 取得した stdout/stderr 全文 |

### 10.4 引数組み立て（まとめ）
ダウンロード実行時の組み立て例:
```
novel_downloader.exe <url>
   --output-dir "<保存先>"
   [--use-site-cover | --cover-image "<FILE>"]   # 表紙ラジオに対応（おまかせ時は無し）
   [--horizontal] [--kobo]
   --delay <SEC> --encoding <ENC>
   （env: PYTHONUNBUFFERED=1 ／ flag: CREATE_NO_WINDOW）
```
注: `--no-epub` は v1 では出さない（ePub 生成が目的のため／§5・§14スコープ）。

### 10.5 永続化される設定（前回値の記憶／§5）
保存先・表紙の選択・横書き・Kobo・取得間隔・文字コード を次回起動時に復元。保存場所/形式は §13。

## 11. 先に潰しておくべき注意点

- **ハーメルン（playwright/Chromium 必要）**: Chromium 数百MB の同梱は非現実的。v1 では `--detect-site` の `needs_playwright` を見て**ダウンロード前に早期に弾く**（§7）。将来 playwright 導入時は同フラグで解禁可能。
- **コンソール窓の抑制**: `--windowed` ビルド時、子プロセスに黒窓が出ないよう Windows の `CREATE_NO_WINDOW` を指定。
- **出力バッファリング**: 子プロセスに `PYTHONUNBUFFERED=1` を渡さないとライブ進捗が出ない（§9.3）。
- **既存 `.exe` を呼ぶ構成（①A）**: GUI ランチャーとエンジンを別ビルドにでき、エンジン更新が楽。

## 12. パッケージング方針

### 12.1 構成: 2つの exe を並置（推奨 / Strategy P1）
- **GUI exe**（`--windowed`・CustomTkinter）＋ **既存 CLI exe**（`novel_downloader.exe`・コンソール）の2ファイル。
- GUI は同じフォルダの CLI exe を subprocess 起動（§6・①A）。
- 採用理由:
  - 既存の CLI exe ビルドを**そのまま再利用**（無改修・実績あり）。
  - CLI のコンソール出力 UX を壊さない（GUI を `--windowed` にしても CLI は別ファイル）。
  - **独立更新**: 本体（スクレイパー追加）を更新しても GUI を作り直さなくてよい。
  - エンジンの場所解決が自明（GUI exe と同じディレクトリ）。
- 配布は zip かインストーラに2ファイルを同梱し、ユーザーは **GUI exe だけをクリック**。

代替（Strategy P2）: **単一の二刀流 exe**（引数なし→GUI／引数あり→CLI、自分自身を spawn）。
- 長所: 配布が1ファイル（persona に最も優しい）。
- 短所: `--windowed` だと CLI 実行時にコンソール出力が見えない／onefile は自己 spawn ごとに再展開コストがあり `--detect-site` の起動が遅くなる。
- → v1 は **P1 推奨**。単一ファイル配布を重視するなら将来 P2 を検討。

### 12.2 エンジンの場所解決
- 凍結時（PyInstaller）: `os.path.dirname(sys.executable)` 隣の `novel_downloader.exe`。
- 開発時（.py 実行）: `python novel_downloader.py`。
- → `getattr(sys, "frozen", False)` で分岐し、起動方法を1関数に隠蔽する。

### 12.3 依存の同梱
| 依存 | 扱い |
|---|---|
| tkinter / Tcl-Tk | PyInstaller が自動同梱 |
| CustomTkinter | **データ同梱が必要**（テーマ/アセット）。`--collect-data customtkinter`（既知の落とし穴） |
| Pillow | 同梱（JPEG表紙・画像プレビュー用）。数MB |
| requests / beautifulsoup4 | 同梱（小） |
| **playwright / Chromium** | **同梱しない**。`--exclude-module playwright` で除外。GUI はハーメルンを DL 前に弾く（§7.2）ため未使用。外さないと数百MB肥大化 |
| 表紙用日本語フォント | Pillow 表紙にはフォントが必要。Windows では `font/` 同梱フォント or システムフォントへのフォールバックを要確認（TODO・§15） |

### 12.4 サイズ目安
Chromium を除けば、Python ランタイム＋Tcl/Tk＋Pillow＋CustomTkinter で **概ね 30〜50MB**。Chromium 同梱なら 300MB+（回避）。

### 12.5 ビルド時の要点
- ビルドは **Windows 上**で行う（PyInstaller はクロスコンパイル不可。開発は WSL2 でも exe 生成は Windows Python で）。
- GUI 例: `pyinstaller --windowed --onefile --collect-data customtkinter --exclude-module playwright --icon novel_downloader.ico gui.py`
- CLI: 既存のビルド手順を踏襲（`WINDOWS_SETUP.md` 参照）。
- **onefile/onedir = v1 は onefile で確定**（単一ファイルの簡便さを優先。persona は起動待ちを許容、PC慣れユーザーは CLI を使う想定）。CLI exe が onefile だと `--detect-site` 等の spawn ごとに ~1秒の再展開コストがあるが許容範囲。**AV 誤検知が実際に起きたら onedir へ切替**を検討。

### 12.6 配布時の摩擦（persona 向けに要注意）
- **SmartScreen 警告**: 署名なし exe は「WindowsによってPCが保護されました／発行元不明」が出る。非技術者には怖い。
  - **v1 方針 = 無署名＋手順案内（②）**: コード署名は見送り。同梱 README に
    **「詳細情報 → 実行」** と、右クリック→プロパティ→**「ブロックの解除」** の手順を**スクリーンショット付き**で記載。
  - 補足: EV 証明書は年7〜8万円で費用対効果が悪く見送り。標準(OV)証明書も即時信頼が付かず微妙。
    将来、不特定多数へ広く配る／警告ゼロにしたい場合は **Azure Trusted Signing（月約$10）** を第一候補に検討（個人の利用資格は要確認）。
  - 特定少数（自分・知人）配布なら、OneDrive/USB 経由で渡せば MOTW が付かず警告自体を回避できる（既存 `forwindows/` ワークフロー活用）。
- **ウイルス対策の誤検知**: PyInstaller exe はまれに AV に誤検知される。
  - 軽減: **`--noupx`**（UPX圧縮無効）。それでも誤検知が起きたら **onedir へ切替**＋各AVベンダーへ検体提出して許可登録（v1 は onefile 採用・§12.5）。
- **配布形式**: v1 は zip（GUI exe＋CLI exe＋簡易 README）で十分。Start メニュー登録等が欲しければ Inno Setup / NSIS のインストーラは将来。

### 12.7 更新運用
- 本体（スクレイパー）更新時は CLI exe を差し替えるだけ（P1 の独立更新の利点）。
- persona には zip 再配布が最も単純。自動更新は v1 スコープ外。

### 12.8 アプリアイコン
- 同梱アイコン: **`novel_downloader.ico`**（プロジェクトルート。現状 16×16 / 32×32 を収録）。
- exe アイコン: `pyinstaller … --icon novel_downloader.ico`。
- ウィンドウ／タスクバー: CTk ルートで `self.iconbitmap(resource_path("novel_downloader.ico"))`（Windows）。
- リソース解決: onefile は実行時に `sys._MEIPASS` へ展開されるため、`--add-data "novel_downloader.ico;."` で同梱し、
  `resource_path()`（frozen 判定で `sys._MEIPASS` or スクリプト隣を返す）経由で絶対パス化する。
- 推奨（任意）: Explorer の大アイコン／高DPI で滲まないよう **48×48 と 256×256 を追加**した .ico が理想（現状は 16/32 のみ）。

## 13. 設定の永続化

### 13.1 保存場所
- **`%APPDATA%\NovelDownloader\settings.json`**（`os.environ["APPDATA"]`）。
- **exe の隣に置かない**: PyInstaller onefile は一時ディレクトリに展開されるため、隣に書くと消える／書けない。ユーザー毎の `%APPDATA%` が正解。
- 開発時（Linux）フォールバック: `~/.config/novel_downloader_gui/settings.json`。

### 13.2 形式（JSON・schema 付き）
```json
{
  "schema": 1,
  "output_dir": "C:\\Users\\xxx\\Downloads\\小説",
  "cover_mode": "auto",          // "auto" | "site" | "file"
  "cover_image_path": "",        // cover_mode=="file" のときのみ意味を持つ
  "horizontal": false,
  "kobo": false,
  "toc_at_end": false,
  "font_path": "",            // 埋め込みフォント。空＝標準（埋め込みなし）
  "delay": 1.5,
  "encoding": "utf-8"
}
```
`schema` を持たせ、将来のキー追加・移行に備える（`--detect-site`／`--list-sites` と同方針）。

### 13.3 読み書きの規則
| 局面 | 挙動 |
|---|---|
| 読み込み（起動時） | ファイルが無い／壊れている→**既定値で起動**（クラッシュしない）。欠損キーは既定で補完（前方互換） |
| `schema` 不一致 | 解釈不能なら既定にリセット（or 移行）。例外で落とさない |
| 書き込み | 設定変更のたびに保存（cheap）。**アトミック書き込み**（`tempfile.mkstemp`+`os.replace`、本体 `_save_watch_cache` と同方式） |
| 妥当性 | `output_dir` 不在→起動時に作成 or 既定へ。`cover_image_path` 不在→`cover_mode` を `"auto"` にフォールバック（本体仕様と整合） |

### 13.4 既定値
| キー | 既定 |
|---|---|
| `output_dir` | `%USERPROFILE%\Downloads\小説`（実行時に解決） |
| `cover_mode` | `"auto"` |
| `cover_image_path` | `""` |
| `horizontal` / `kobo` | `false` |
| `toc_at_end` | `false` |
| `font_path` | `""`（標準・埋め込みなし） |
| `delay` | `1.5` |
| `encoding` | `"utf-8"` |

### 13.5 任意（v1 では見送り可）
- ウィンドウのサイズ／位置の記憶。
- 詳細設定の開閉状態の記憶。
